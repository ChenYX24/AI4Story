## 2. 模块一：用户认证 (Auth)

**涉及文件：**
- Router: `apps/api/routers/auth.py`
- 数据库: `apps/api/db.py`
- 前端 Store: `apps/web/src/stores/user.ts`

**流程图：**

```
┌──────────┐    POST /api/auth/register     ┌──────────────┐
│  前端     │ ──────────────────────────────▶ │  auth.py      │
│  用户表单  │    {nickname, password}         │               │
└──────────┘                                 │  1. 检查昵称唯一│
                                             │  2. SHA256 加盐 │
     ┌───────────────────────────────────────│  3. 生成 token  │
     │         {user_id, token}              │  4. 写入 SQLite │
     ▼                                       └──────────────┘
┌──────────┐
│ localStorage │  存储 mindshow_token
│ (前端持久化)  │  后续请求自动注入 Bearer Header
└──────────┘

登录流程:
  POST /api/auth/login → 校验密码 → 刷新 token → 返回用户信息

会话校验:
  GET /api/auth/me → 从 Header 取 token → db.user_by_token() → 返回用户
```

**数据模型 (SQLite)：**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | TEXT PK | UUID |
| nickname | TEXT UNIQUE | 用户昵称 |
| password_hash | TEXT | SHA256(salt + password) |
| token | TEXT | Bearer token (secrets.token_hex) |
| created_at | INTEGER | 创建时间戳 |
| last_login_at | INTEGER | 最后登录时间戳 |

## 3. 模块二：故事加载 (Story / Scene)

**涉及文件：**
- Router: `apps/api/routers/story.py`, `apps/api/routers/stories.py`
- 加载器: `apps/api/scene_loader.py`
- 前端 Store: `apps/web/src/stores/story.ts`

**故事数据结构：**

```
scenes/
├── story_scenes.json              ← 主故事元数据 (标题、角色列表、场景索引)
├── 000/                           ← 全局资产
│   ├── global_character/*.png     ← 角色立绘 (透明背景)
│   └── global_object/*.png        ← 道具图片
└── 001/ ~ 010/                    ← 各场景
    ├── scene.json                 ← 场景元数据 (类型、背景、角色、对话)
    ├── background.png             ← 场景背景
    ├── scene_character/*.png      ← 场景特定角色变体
    ├── scene_object/*.png         ← 场景特定道具
    └── comic.png                  ← 叙事场景漫画图
```

**加载流程图：**

```
┌──────────┐  GET /api/story?id=xxx   ┌────────────────────┐
│  前端     │ ────────────────────────▶│  story.py           │
└──────────┘                          │                    │
                                      │  scene_loader.py   │
                                      │  ┌────────────────┐│
                                      │  │ 1. 读取         ││
                                      │  │ story_scenes.json││
                                      │  │ 2. 构建摘要     ││
                                      │  │ 3. 解析角色/道具 ││
                                      │  └────────────────┘│
                                      └────────┬───────────┘
                                               │
                                      ┌────────▼───────────┐
                                      │  返回 StoryPayload  │
                                      │  · title, summary   │
                                      │  · characters[]     │
                                      │  · objects[]        │
                                      │  · scene_count      │
                                      └────────────────────┘

┌──────────┐  GET /api/scene/{idx}    ┌────────────────────┐
│  前端     │ ────────────────────────▶│  scene_payload()    │
└──────────┘                          │                    │
                                      │  1. 读取 scene.json │
                                      │  2. 判断场景类型:    │
                                      │     · narrative     │
                                      │     · interactive   │
                                      │  3. 解析角色 URL:    │
                                      │     场景级 → 全局级  │
                                      │  4. 返回完整场景数据  │
                                      └────────────────────┘
```

**场景类型：**

| 类型 | 说明 | 用户行为 |
|------|------|---------|
| `narrative` | 叙事场景 | 只读，展示漫画 + 旁白 + 对话 |
| `interactive` | 互动场景 | 拖拽角色/道具，构建操作，触发 AI 生成 |

## 4. 模块三：互动叙事生成 (Interact → Narrative)

**涉及文件：**
- Router: `apps/api/routers/interact.py`
- 核心服务: `apps/api/services/narrative_generator.py`
- 前端组件: `apps/web/src/components/InteractiveView.vue`
- 前端 Store: `apps/web/src/stores/interact.ts`

**这是系统最核心的 Pipeline，分三步完成：**

```
┌─────────────────────────────────────────────────────────────────────┐
│                    互动叙事生成 Pipeline (30-90s)                     │
└─────────────────────────────────────────────────────────────────────┘

  用户操作 (前端)
  ┌─────────────────────────────────────────┐
  │ 1. 拖拽角色/道具到画布上                   │
  │ 2. 选择两个对象 (subject + target)        │
  │ 3. 输入动作描述 (如 "把篮子给奶奶")        │
  │ 4. 构建 Operation: {subject, target, action} │
  │ 5. 点击 "完成"                            │
  └──────────────────┬──────────────────────┘
                     │
                     ▼ POST /api/interact
  ┌──────────────────────────────────────────┐
  │ InteractRequest                          │
  │ · story_id, session_id, scene_idx        │
  │ · placements: [{id, transform}]          │
  │ · ops: [{subject, target, action}]       │
  │ · custom_props: [{name, url}]            │
  └──────────────────┬───────────────────────┘
                     │
    ═══════════════════════════════════════════
    ║          Step 1: Qwen LLM 叙事生成       ║
    ═══════════════════════════════════════════
                     │
                     ▼
  ┌──────────────────────────────────────────┐
  │ 构建 Prompt:                              │
  │ · 故事背景 + 当前场景描述                   │
  │ · 角色性格/动机约束                        │
  │ · 用户操作序列 (ops)                       │
  │ · 自定义道具描述                           │
  │                                          │
  │ 调用 DashScope Qwen API                   │
  │                                          │
  │ 输出 JSON:                                │
  │ · summary: 情节摘要                       │
  │ · narration: 旁白文本                     │
  │ · dialogue: [{speaker, line}]            │
  │ · storyboard: [panel1, panel2, panel3, panel4] │
  └──────────────────┬───────────────────────┘
                     │
    ═══════════════════════════════════════════
    ║       Step 2: 参考图板 (Reference Board)  ║
    ═══════════════════════════════════════════
                     │
                     ▼
  ┌──────────────────────────────────────────┐
  │ 收集视觉素材:                              │
  │ · 场景背景 background.png                  │
  │ · 参与操作的角色 PNG                       │
  │ · 参与操作的道具 PNG                       │
  │ · 自定义道具 PNG                           │
  │                                          │
  │ 合成参考图板 → _refboard.png               │
  └──────────────────┬───────────────────────┘
                     │
    ═══════════════════════════════════════════
    ║       Step 3: Seedream 四格漫画生成       ║
    ═══════════════════════════════════════════
                     │
                     ▼
  ┌──────────────────────────────────────────┐
  │ 构建 Seedream Prompt:                     │
  │ · 4 个分镜描述 (来自 Step 1)               │
  │ · 参考图板 (来自 Step 2)                   │
  │                                          │
  │ 调用 Volcengine Ark API                   │
  │ · 模型: Seedream                          │
  │ · 输出: 1920×1920 PNG (2×2 四格)          │
  │                                          │
  │ 后处理:                                   │
  │ · 生成缩略图 256×256 JPEG                  │
  │ · 保存 node.json 元数据                    │
  └──────────────────┬───────────────────────┘
                     │
                     ▼
  ┌──────────────────────────────────────────┐
  │ InteractResponse                         │
  │ · node_id: 动态节点 ID                    │
  │ · summary, narration, dialogue           │
  │ · storyboard: [4 panel descriptions]     │
  │ · comic_url: /outputs/.../panel.png      │
  │ · thumbnail_url: /outputs/.../thumb.jpg  │
  └──────────────────────────────────────────┘
                     │
                     ▼ 前端
  ┌──────────────────────────────────────────┐
  │ 1. 将动态节点插入 story flow               │
  │ 2. 展示四格漫画                            │
  │ 3. 预加载 TTS 语音                        │
  │ 4. 记录交互到 interactions 历史            │
  │ 5. 启用 "下一页" 按钮                     │
  └──────────────────────────────────────────┘
```

**输出文件结构：**

```
outputs/webdemo/{session_id}/dynamic/{node_id}/
├── panel.png        ← 1920×1920 四格漫画
├── thumb.jpg        ← 256×256 缩略图
├── node.json        ← 元数据 (summary, narration, dialogue, storyboard)
└── _refboard.png    ← 参考图板 (调试用)
```

## 5. 模块四：自定义道具生成 (Prop)

**涉及文件：**
- Router: `apps/api/routers/create_prop.py`
- 服务: `apps/api/services/prop_generator.py`
- 前端组件: `apps/web/src/components/CustomPropCreateModal.vue`
- 前端组件: `apps/web/src/components/SketchPadModal.vue`

**三种生成模式：**

```
┌─────────────────────────────────────────────────────────────────┐
│                    道具生成 Pipeline                              │
└─────────────────────────────────────────────────────────────────┘

  模式 A: AI 生成 (默认)
  ─────────────────────
  用户输入名称 + 描述
       │
       ▼
  POST /api/create_prop
  {name: "魔法帽", description: "紫色尖顶巫师帽"}
       │
       ▼
  ┌────────────────────────┐
  │ Seedream 生成           │
  │ Prompt: "Create one    │
  │ single '{name}' object │
  │ illustration..."       │
  └──────────┬─────────────┘
             │
             ▼
  ┌────────────────────────┐
  │ rembg 去背景            │
  │ 输入: RGB PNG           │
  │ 输出: RGBA 透明 PNG     │
  └──────────┬─────────────┘
             │
             ▼
  保存: outputs/{session}/custom_props/{slug}.png


  模式 B: 跳过 AI (skip_ai=true)
  ──────────────────────────────
  用户上传照片/涂鸦 → 直接作为道具使用
  无 Seedream 调用，仅存储


  模式 C: 批量生成
  ────────────────
  POST /api/create_props_batch
  多个道具一次生成 → 网格图 → 切割单元格 → 各自去背景


  模式 D: 智能生成
  ────────────────
  POST /api/create_props_smart
  根据场景上下文自动推荐并生成道具
```

**前端交互流程：**

```
  ┌──────────────┐     ┌──────────────┐     ┌──────────────┐
  │  上传照片     │     │  拍照         │     │  手绘涂鸦     │
  │  (文件选择)   │     │  (摄像头)     │     │ (SketchPad)  │
  └──────┬───────┘     └──────┬───────┘     └──────┬───────┘
         │                    │                    │
         └────────────────────┼────────────────────┘
                              │
                    ┌─────────▼─────────┐
                    │ CustomPropCreate   │
                    │ Modal              │
                    │ · 输入名称         │
                    │ · 输入描述         │
                    │ · 选择: AI生成     │
                    │   或 直接使用      │
                    └─────────┬─────────┘
                              │
                    ┌─────────▼─────────┐
                    │ 调用 API 生成      │
                    │ 返回道具 URL       │
                    │ 添加到画布道具栏   │
                    └───────────────────┘
```

## 8. 模块七：成长报告 (Report)

**涉及文件：**
- Router: `apps/api/routers/report.py`
- 服务: `apps/api/services/report_service.py`
- 前端页面: `apps/web/src/pages/ReportPage.vue`
- 前端 Composable: `apps/web/src/composables/useReportStream.ts`

**报告包含三个板块：**

| 板块 | 受众 | 内容 |
|------|------|------|
| Share (分享) | 公开 | 荣誉称号、成就徽章、故事摘要 |
| Kid (孩子) | 儿童 | 你的故事 vs 原版故事、差异点、思考题 |
| Parent (家长) | 家长 | 能力维度评分、性格特征、成长建议 |

**SSE 流式生成流程图：**

```
┌─────────────────────────────────────────────────────────────────┐
│                    成长报告 SSE Pipeline                          │
└─────────────────────────────────────────────────────────────────┘

  POST /api/report/stream
  ReportRequest: {session_id, story_id, interactions[]}
       │
       ▼
  ══════════════════════════════════════
  ║  Stage A: 分析交互 (analyze)        ║
  ══════════════════════════════════════
       │
       ├──▶ SSE: {stage: "analyze", status: "running"}
       │
       ├──▶ 逐场景分析:
       │    ├── SSE: {type: "per_scene", idx: 1, total: N}
       │    ├── SSE: {type: "per_scene", idx: 2, total: N}
       │    └── ...
       │
       ├──▶ SSE: {stage: "analyze", status: "done"}
       │
       ▼
  ══════════════════════════════════════
  ║  Stage B: 生成报告 (compose)        ║
  ══════════════════════════════════════
       │
       ├──▶ SSE: {stage: "compose", status: "running"}
       │
       ├──▶ 调用 Qwen LLM (线程池)
       │    · 输入: 所有交互数据 + 故事上下文
       │    · 输出: 三板块结构化 JSON
       │    · 耗时: 2-10 秒
       │
       ├──▶ SSE: {stage: "compose", status: "done"}
       │
       ▼
  ══════════════════════════════════════
  ║  Stage C: 流式推送 (stream chunks)  ║
  ══════════════════════════════════════
       │
       ├──▶ SSE: {chunk: "share"}
       │    · 荣誉称号、成就列表
       │
       ├──▶ SSE: {chunk: "kid_header"}
       │    · 你的故事 vs 原版
       │
       ├──▶ SSE: {chunk: "kid_list"}
       │    · 差异点、思考题
       │
       ├──▶ SSE: {chunk: "parent_metrics"}
       │    · 能力维度: 想象力/共情/解决问题/...
       │    · 每项 1-10 分 + 具体行为引用
       │
       ├──▶ SSE: {chunk: "parent_lists"}
       │    · 性格特征、待提升点、观察记录、建议
       │
       ▼
  ══════════════════════════════════════
```
